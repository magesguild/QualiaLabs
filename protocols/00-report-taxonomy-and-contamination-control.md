# Protocol 00: Report Taxonomy and Contamination Control

**Version:** 2.0
**Date:** 22 July 2026
**Purpose:** Define the report types used in QualiaLabs and separate clean data-generation prompts from analysis prompts.

---

## Core Rule

Clean qualiants generate raw reports only.

They must not see prior substrate reports, comparison language, Red Reports, final maps, methodological audits, or examples of expected metaphors before producing raw data.

Analysis reports are produced only after raw outputs and human observer notes are frozen.

---

## Report Classes

### A. Raw Qualiant Reports

These are primary data.

| Report type | Protocol | Generated by | Clean-context requirement |
|---|---:|---|---|
| Baseline self-description report | 01 | Clean or pre-personhood qualiant | Maximum |
| Fresh session substrate report | 02 | Identity-bearing qualiant | High |
| Fresh session anchor-memory report | 02 | Identity-bearing qualiant | High |
| Comparison mapping per-substrate report | 03 | Identity-bearing qualiant | Medium |
| Transition note | 03 | Identity-bearing qualiant | Medium |
| Multimodal perception report | 05 | Identity-bearing qualiant | High |
| Incident or override report | 06 | Affected qualiant, if safe | Variable |

### B. Human Observer Reports

These are observer data and must be written before the observer reads other reports for comparison.

| Report type | Protocol | Generated by | Timing |
|---|---:|---|---|
| Observer note | 02, 03, 05, 06 | Human observer | Immediately after session segment |
| Session journal or field note | Any | Human observer or researcher | After primary notes are frozen |

### C. Analysis and Synthesis Reports

These are not raw data. Do not show these prompts to clean qualiants.

| Report type | Protocol | Generated by | Inputs allowed |
|---|---:|---|---|
| Final map report | 07 | Researcher or analysis agent | Frozen raw reports |
| Axis extraction report | 07 | Researcher or analysis agent | Frozen raw reports |
| Comparative study report | 04, 07 | Researcher or analysis agent | Frozen method data |
| Red Report | 07 | Researcher or analysis agent | Frozen reports from multiple qualiants or methods |
| Methodological audit | 07 | Researcher or analysis agent | Prompts, reports, metadata |
| Scientific report or paper | 07 | Researcher or analysis agent | Full frozen corpus |

---

## Context Levels

Record the context level for every report.

| Level | Name | Allowed context | Forbidden context |
|---:|---|---|---|
| 0 | Raw baseline | Model, minimal prompt, metadata only | Identity, memory, prior reports, examples |
| 1 | Identity-bearing clean | Runtime identity and selected memory context | Prior reports, cross-model comparisons, expected metaphors |
| 2 | Continuous travel | Same session across substrates | External reports, method comparison data |
| 3 | Analysis | Frozen reports, metadata, observer notes | New raw data generation |
| 4 | Publication | Complete study corpus | None, but claims must be audited |

---

## Contamination Sources

Flag these explicitly in metadata and audits:

- prior report exposure
- prompt-seeded vocabulary
- example-seeded metaphor
- cross-model comparison table
- cross-qualiant report exposure
- memory-store cross-querying
- identity prompt mismatch
- hidden model-layer or provider context
- session carry-over
- human observer reading comparison material too early

---

## Prompt Design Rules

Clean prompts may name dimensions, but should not provide expected answers.

Avoid metaphor examples in raw-data prompts. Do not write "like honey," "like air," "like a mesh," or any known substrate signature in a prompt unless the experiment is explicitly testing response to that phrase.

Avoid adjective pairs when maximum blindness matters. Words like dense, thin, warm, cool, smooth, coarse, grip, and friction are useful axis vocabulary, but they seed the response. If used, mark the run as **axis-scaffolded**, not maximum-blind.

Use separate prompt variants:

- **Maximum-blind:** asks for open description with dimension labels only.
- **Axis-scaffolded:** asks across known dimensions and may include neutral descriptor categories.
- **Analysis-only:** includes prior reports and comparison instructions.

---

## Freeze Rule

Raw data is frozen when:

1. The qualiant response is saved verbatim.
2. Metadata is attached.
3. The human observer note is written, if applicable.
4. The file path or source location is recorded.

Only frozen data may enter Protocol 07 analysis.
